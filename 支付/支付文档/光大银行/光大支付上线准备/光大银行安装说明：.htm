一、光大银行支付环境配置的准备工作：
1、测试报告通过后，光大会给商户发来公钥：见附件（正式环境证书文件夹下的CEP_PR_PUBKEY.cer）
公钥的CN值：041@81000001001174@中国光大银行股份有限公司@00000001
 2、正式签约光大支付后，会收到一个两码证书，用两码证书上给的参考号和授权码下载的私钥（正式环境证书文件夹下的benlaiCEBPrivateKey.pfx）
私钥的CN值：041@82008854143@2008854143@00000003
备注：“CN = 041****”中等号左右有空格不能直接复制使用，其次如果商户的CN中间有空格，需要用英文的问号代替  
3、怎样查看公钥活私钥的CN值
公钥的CN值的查看，双击公钥CEP_PR_PUBKEY.cer，详细信息>点击使用者，就能看到CN值。
私钥的CN值查看，开始 → 运行 → certmgr.msc ，找到刚安装的证书并双击查看CN。或者在导出私钥的pc上，IE>工具>Internet选项>内容>证书
找到用两码证书导出的私钥，双击详细信息>点击使用者，就能看到CN值。
二、配置服务器的支付环境
1、将附件里safesign文件夹拷贝到C盘里，该文件夹里为光大支付安全服务包，包含①CSIISafeSignForCEB.exe，②port.cfg，③safesignforceb.cfg三个文件，缺一不可。
其中②port.cfg文件里写入安全服务安装的端口号。
③safesignforceb.cfg写入以下配置信息：
51989
CN=041@82008854143@2008854143@00000003
CN=041@81000001001174@中国光大银行股份有限公司@00000001
 （备注：以上配置信息说明
第一行为端口号跟②port.cfg里的端口号必须保持一致。
第二行设置的是商户用于签名的商户私钥CN，=左右不能有空格
第三行设置的是商户用于验签的银行公钥CN，=左右不能有空格  ）
2、将公钥和私钥拷贝至服务器，进行安装，商户私钥安装到个人库，密码 benlai2015。银行公钥安装到受信任的根证书颁发机构库。
（安装证书操作）
3、SAFESIGN.dll拷贝至服务器
32位系统放置到$WINDIR\WINDOWS\system32
64位系统放置到$WINDIR\WINDOWS\SysWOW64 
开始 → 管理员运行 → cmd  进行SAFESIGN.dll所在文件夹，执行SAFESIGN.dll
4、安装安全服务
（1）检查c:/safesign目录下是否有CSIISafeSignForCEB.exe文件。
（2）dos下进入c:/safesign目录，安装服务。
如果不是域用户，则在安装指令中直接输入用户名和用户口令安装服务：
CSIISafeSignForCEB install 用户名 用户口令
如果是使用域用户，则在安装的时候不要输入用户名和用户口
CSIISafeSignForCEB install
（3）安装完成后，
① 找到CSIISafesignforceb服务(右键我的电脑→管理→服务与应用程序→服务)，将 该服务的“启动类型”设成“自动”；
② 打开该服务的属性，在“登录”选项卡中选择“此帐户”并输入域用户的全名及 其密码即可。 
③ 启动CSIISafesignforceb服务。 
5、将私钥（benlaiCEBPrivateKey.pfx）当公钥导出一份(导出时，选择“不，不要导出私钥”，下一步，选择“Base64编码X.509”），具体操作参照支付测试报告通过后光大发的《电子商户操作流程》，
商户财会人员用网盾登录到对公网银，上传公钥。
三、其他
光大支付，若支付参数里包含中文参数，光大要求gbk编码。商户需要将商户支付提交的页面设置为gbk页面。
在IE中，打开商户提交支付的页面，右键，选择“编码”，页面为关闭关闭gb2312,此时包含中文参数，提交到光大，就没有问题，若不是关闭gb2312，提交后，无法跳转到光大支付页面，因为验签失败。
